Features

Completely distributed design

We've designed Buddycloud so that anyone can run their own buddycloud server. This means that companies and privacy aware individuals can maintain their own communication cloud behind the firewall and selectively choose what data they wish to share with others.

The old way of building communication services was to put all the data into a central core.

Buddycloud is a revolutionary new design where anyone can run their own Buddycloud server, and each buddycloud server will find, connect and share the data you choose with users on other Buddycloud servers.

This approach has huge benefits, especially for industries that care about data privacy (Finance, Medicine).

Scalable

Scaling your service because it's popular is a 'good problem to have'. And it needs a good roadmap: so we have designed Buddycloud to grow with you and scale up without bottlenecks. Buddycloud is currently tested against servers that scale to over 10M simultaneous users. Additionally, each server can subsequently be scaled horizontally to support more simultaneous users.

Buddycloud also works with commercial realtime CDNs like fanout.io which help you offload capacity to a Global push CDN.

Rich Communications

Move beyond just text sharing: Buddycloud enables you to share media files with just one channel or with the entire world. And to make development easy, the Buddycloud API will return scaled thumbnails for easy embedded into your application.

Our rich communication roadmap includes adding full webRTC video calling between Buddycloud channels.

By developers, for developers

Buddycloud is designed make adding communication to web and app development as simple as possible by offloading as much of the tricky application logic to the server. This means you can quickly write reliable, services by just using the existing Buddycloud libraries.

The Buddycloud platform follows a convention over configuration approach that helps you quickly build without pre-configuration. {action} view the API {action}

Open source and open standards

Secure communication needs open, inspectable secure communications come from publishing code as open source. Buddycloud's code is maintained on Github and released under the Apache 2.0 license. Buddycloud builds on existing standards like ActivityStrea.ms and the ATOM syndication format: you benefit from being able to reuse a wide range of existing libraries.

Buddycloud works closely with the W3C, Mozilla Foundation, and XSF to build the strong foundation for an open communication ecosystem.

Secure by design.

Buddycloud is used by security-sensitive agencies to protoct their communications.

Buddycloud's modular design means that components communicate in their own security space and leverage the tested XMPP communication security for inter-module communication.

All code is peer-reviewed and extensively tested.

All communication between buddycloud servers and clients uses TLSv1.2 for encryption.

Buddycloud uses hashed passwords

User privacy

Buddycloud enables users to set different privacy settings per channel. Administrators can set permissions per Buddycloud site. Users can select who can view their channel, make it open to just other members of their Buddycloud site, or open to the world.

Run it yourself or have Buddycloud host it for you.

Buddycloud comes in two compatible flavours: fully hosted and an install-it-yourself set of packages and instructions. Both editions use exactly the same code so that you have the flexibility to migrate your data and users between platforms by just changing the API url.

We recommend you test drive Buddycloud using the existing public API on the hosted service before spinning up your own install. Then when you are ready, follow the Install Guide or Sign up for professional hosting.

Community and enterprise editions

The Buddycloud community edition is open source, free to download and run on your site. It aims to provide the best both worlds: rapid updates and fixes with long-term stability, supportability, and maintainability.

The Buddycloud enterprise edition is integrated, tested, and Includes patches, updates, SLA-based support, multi-year maintenance policies.

Modular

Keep your operations lean and mean by only running the Buddycloud services you need. If you don't need to support media or push notifications, it's as simple as not installing these services. If you find out that you later need social recommendations, just add that Buddycloud component.

Easy to use

As a general rule we don't want to invent an enormous system that takes ages to master. Buddycloud should be as easy to use as reading the API and getting data from endpoints like api.example.com/testuser@myapp.com/content/posts.
